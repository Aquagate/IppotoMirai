# IppotoMirai

「今日の一歩」ダッシュボード（表）と、「未来の一歩」生成（裏）を、GitHub Pages 上で動かす実験プロジェクト。  
データは **GitHubには保存しない**（Public運用前提）。

---

## 使い方（最短）

### 1) 今日の一歩（表）
- 入口：`/`（トップページ）
- 目的：今日の一歩を入力・保存・閲覧する

### 2) 未来の一歩（裏）
- 入口：`/future/`
- 目的：未来側の提案・検討・生成（負荷の高い処理はここに隔離）

※ 表⇄裏は同じ保存領域（localStorage）を参照する想定。

---

## 公開URL（GitHub Pages）
- 表：`https://<USER>.github.io/<REPO>/`
- 裏：`https://<USER>.github.io/<REPO>/future/`

> `<USER>` と `<REPO>` は各自の環境に合わせて読み替え。

---

## 保存とデータ取り扱い（重要）

### 保存先
- 基本：ブラウザの `localStorage`（同一ドメイン配下で共有される）
- 将来：OneDrive（Microsoft Graph）連携も想定（設定はUI入力で保持）

### Public運用ルール（絶対）
- **実ログ（個人の行動ログ）をリポジトリにコミットしない**
- **トークン／キー／クライアントシークレット等をコードに埋め込まない**
- テストに使うのは **ダミーデータのみ**（個人情報禁止）

※ 公開リポジトリはコミット履歴も公開対象。誤って入れると「消して終わり」にならない。

## 開発メモ（運用方針）
- 巨大な単一HTMLは廃止し、マルチファイル化（方式A）で保守性を確保する
- 「重い処理」は裏（/future）に寄せ、表（/）の軽さを守る
- 画面が白紙になった場合は、まず Pages 上（https）で再現確認する  
  （`file://` 直開きはブラウザ制約でJSや認証系が死ぬことがある）
